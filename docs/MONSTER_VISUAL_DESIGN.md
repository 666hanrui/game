# 怪物与种族视觉辨识设计说明

本轮目标不是增加新玩法，而是解决局内小尺寸下的识别问题：玩家要能一眼看出自己是什么种族、敌人是什么类型、怪物大致威胁是什么。

## 一、种族 sprite 设计原则

种族资源位于：

```text
public/assets/sprites/races/
```

### 人族 human

定位：标准、均衡、最像“人”。

视觉关键词：

- 明确头部、身体、手臂、腿部。
- 蓝色主色对应人族 UI 色。
- 比旧版圆球更像一个角色，避免玩家误以为只是蓝色棋子。

### 精灵 elf

定位：高敏、轻盈、风筝型。

视觉关键词：

- 大尖耳是第一识别点。
- 身体更瘦、更高。
- 两侧浅绿色风痕强化“移动快、轻盈”。

### 哥布林 goblin

定位：小、狡猾、前期强势。

视觉关键词：

- 身体比其他种族更矮、更圆、更小。
- 尖耳外扩，眼睛偏锐利。
- 轮廓偏低趴，和兽人的宽大厚重区分开。

### 兽人 orc

定位：厚重、强壮、硬顶。

视觉关键词：

- 宽肩、粗手臂、大体型。
- 角和獠牙强化野蛮感。
- 橙红色大块面保证小尺寸下也足够醒目。

### 灵族 spirit

定位：半透明、漂浮、后期成长。

视觉关键词：

- 身体下方做成幽灵状波浪轮廓。
- 使用半透明紫色和外溢光痕。
- 底部阴影弱，体现漂浮感。

## 二、怪物基础轮廓

怪物资源位于：

```text
public/assets/sprites/enemies/
```

### 史莱姆 slime

定位：普通软体怪。

轮廓：

- 软塌大 blob 形状。
- 边缘不规则，底部更宽。
- 靠“软体轮廓”识别，不只靠绿色。

### 蜘蛛 spider

定位：高压近身怪。

轮廓：

- 六边形主体。
- 多足外扩，远看像带刺的多腿怪。
- 和史莱姆的圆润轮廓、骷髅的尖锐轮廓明显区分。

### 骷髅 skeleton

定位：偏硬、偏尖锐的敌人。

轮廓：

- 外层菱形。
- 内部 skull 结构和黑眼洞。
- 远看是白灰色尖锐目标，不再只是普通白块。

## 三、怪物角色 EnemyRole 的局内标记

当前代码中的角色类型：

```ts
basic | fast | tank | ranged | elite | boss
```

本轮没有新增怪物类型，没有改刷怪逻辑，只在 `src/entities/Enemy.ts` 的渲染层增加了角色轮廓标记。

### basic 普通怪

- 只使用基础怪物 sprite。
- 不额外加符号，降低视觉噪音。

### fast 快怪

- 左右两侧加小箭形轮廓。
- 即使颜色看不清，也能靠“推进箭头”判断它移动快。

### tank 坦克怪

- 外层方形装甲框。
- 横竖中线模拟加固装甲。
- 目标是让玩家一眼判断“它很硬”。

### ranged 远程怪

- 外层瞄准环和四向准星。
- 目标是让玩家提前意识到它会打远程弹。

### elite 精英怪

- 上方和左右加尖刺标记。
- 目标是区别于普通大怪，不只靠红色光环。

### boss

- 外层大圆环 + 顶部王冠轮廓。
- 目标是在混战中保持最高优先级识别。

## 四、后续建议

如果后面要继续做更细的怪物美术，建议不要直接在主循环里新增逻辑，可以先拆成两层：

```text
EnemyType：slime / spider / skeleton / future_xxx
EnemyRole：basic / fast / tank / ranged / elite / boss
```

资源映射可以进一步扩展为：

```ts
enemySprite[type][role]
```

但当前阶段先用“基础 sprite + role 轮廓标记”的方式，改动小、和主开发冲突少，也能马上解决局内辨识度问题。
