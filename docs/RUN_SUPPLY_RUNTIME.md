# 局内补给系统实现说明

本文件说明当前局内补给系统的实现方式。

## 当前已接入道具

当前已实装 12 个局内补给：

- 磁铁
- 护盾
- 急速药剂
- 攻击药剂
- 血包
- 回春露
- 暴击药剂
- 冰霜炸弹
- 雷击符石
- 地裂符石
- 诱饵人偶
- 机械炮台

这些补给来自 `src/data/runItems.ts` 的局内补给定义。

重要边界：局内补给只服务单局战斗节奏，不作为宝箱主奖励。

说明：`src/data/runItems.ts` 中还存在一些预埋补给，例如聚宝符、净化符、狂暴药剂、佣兵号角、亡骨号角、丰收印记、猎魂灯等；这些暂未进入 `RuntimeSupplyId`，不属于当前运行时刷新池。检查 ID 是否遗漏时，应以“已经写入 `RuntimeSupplyId` 且已有 activate / pickSpawnId / colorFor / labelFor 处理”为准。

## 实现文件

### src/systems/RunSupplyRuntime.ts

局内补给运行时系统。

职责：

- 随时间在玩家附近刷新补给。
- 击杀敌人时低概率刷新补给。
- Boss 击杀后额外刷新补给。
- 渲染补给掉落物。
- 处理玩家拾取补给。
- 管理补给持续时间。
- 渲染补给状态条。
- 渲染磁铁、护盾、回春、暴击等光环。
- 处理即时补给效果，例如血包、冰霜炸弹、雷击符石、地裂符石。
- 处理临时场上装置，例如诱饵人偶和机械炮台。

### src/main.ts

在游戏主循环外层接入 `RunSupplyRuntime`。

调用顺序：

```text
runSupply.beforeGameUpdate()
game.update(dt)
runSupply.afterGameUpdate(dt)
game.render()
runSupply.render(ctx)
```

这种方式的好处是：

- 不直接大改 Game.ts。
- 不直接大改 GameWithSound.ts。
- 不破坏现有战斗逻辑。
- 后续如果补给系统有问题，容易单独回滚。

## 道具效果

### 磁铁

持续 8 秒。

效果：大范围吸取已有经验晶体和普通掉落物。

表现：玩家周围出现蓝色吸附环、虚线范围和向内吸附线。

### 护盾

拾取后获得 1 层护盾，最多 3 层。

效果：抵挡一次受到的伤害。

表现：玩家周围出现浅蓝六边形护盾环，护盾层数以小光点显示；抵挡时出现“护盾抵挡”文字和粒子。

说明：当前护盾通过外层记录 `game.update` 前后的生命变化来抵挡伤害，因此不需要改 CombatSystem。

### 急速药剂

持续 6 秒。

效果：临时降低攻击间隔，提高攻速。

表现：右侧补给状态条显示倒计时，掉落物使用黄色“速”标识。

### 攻击药剂

持续 6 秒。

效果：临时提高玩家伤害。

表现：右侧补给状态条显示倒计时，掉落物使用橙色“攻”标识。

### 血包

即时生效。

效果：立即恢复一段生命值。

定位：基础续航补给，玩家低血量时刷新权重会更高。

表现：绿色“血”标识，拾取后显示恢复数值和治疗粒子。

### 回春露

持续 7 秒。

效果：持续恢复生命。

表现：玩家周围出现绿色回春光环，右侧状态条显示倒计时。

### 暴击药剂

持续 7 秒。

效果：临时提高暴击率和暴击伤害。

表现：玩家周围出现金色暴击弧线，右侧状态条显示倒计时。

### 冰霜炸弹

即时生效。

效果：拾取后立即大幅减速附近敌人。

表现：蓝色“冰”标识，玩家附近爆出冰霜粒子，并显示影响敌人数量。

定位：被围杀时的解围道具。

### 雷击符石

即时生效。

效果：拾取后锁定附近多个高威胁敌人，发射玩家侧能量弹。

表现：紫色雷击粒子，并显示锁定目标数量。

实现说明：雷击符石使用玩家侧 Projectile 处理伤害，这样击杀、经验和掉落仍然走 Game.ts 的正常结算流程。

定位：清杂兵、补刀支援怪、打断压力节奏。

### 地裂符石

即时生效。

效果：拾取后对周围敌人造成击退、减速和破甲，同时释放一圈钝击弹体。

表现：土黄色冲击粒子，并显示破甲影响数量。

实现说明：地裂符石的破甲和击退即时生效，伤害部分通过玩家侧 hammer 弹体处理，尽量避免绕过原本击杀奖励流程。

定位：被围杀时强行打开空间，适合钝器和破甲体系。

### 诱饵人偶

持续 7.5 秒。

效果：部署一个临时诱饵，附近敌人会被诱饵牵引，靠近诱饵时会轻微减速。

表现：橙色人偶装置，周围有吸引范围提示和剩余时间条。

实现说明：诱饵没有改 Enemy.ts 的目标选择，而是在 RunSupplyRuntime 外层对附近敌人的位置进行轻微牵引，降低和怪物主 AI 的冲突。

定位：打乱围杀、给玩家争取走位空间。

### 机械炮台

持续 12 秒。

效果：部署一个临时炮台，自动向附近敌人发射玩家侧能量弹。

表现：蓝色炮台装置，周围有射击范围提示和剩余时间条。

实现说明：炮台发射的是玩家侧 Projectile，击杀和经验结算继续走 Game.ts 的正常流程，不直接在补给系统里手动结算击杀。

定位：机械组职业预埋，提供阵地战和自动火力雏形。

## 补给视觉规则

当前补给掉落物采用“颜色 + 大字 + 简短提示 + 轮廓形状”的方式区分：

```text
磁铁：蓝色菱形，文字“吸 / 吸取”
护盾：浅蓝六边形，文字“盾 / 护盾”
急速药剂：黄色闪电形，文字“速 / 急速”
攻击药剂：橙色三角形，文字“攻 / 攻击”
血包：绿色十字形，文字“血 / 回血”
回春露：绿色圆形，文字“春 / 回春”
暴击药剂：黄色星形，文字“暴 / 暴击”
冰霜炸弹：蓝色雪花形，文字“冰 / 冰霜”
雷击符石：紫色闪电形，文字“雷 / 雷击”
地裂符石：土黄色波纹形，文字“裂 / 地裂”
诱饵人偶：橙色圆靶形，文字“诱 / 诱饵”
机械炮台：蓝色方形，文字“炮 / 炮台”
```

右侧状态条只显示仍在持续的效果。立即生效类补给只显示浮字和粒子，不进入状态条。场上装置类补给会在地图上显示装置本体和剩余时间条。

## 掉落和刷新逻辑

当前补给来源：

- 随时间在玩家附近刷新。
- 击杀敌人时低概率刷新。
- Boss 击杀后额外刷新护盾、药剂和战术装置。
- 玩家低血量时，血包、回春露、护盾、诱饵人偶的刷新权重会提高。

## 暂停和升级阶段处理

当游戏进入升级、选择体系、选择武器、暂停阶段时：

- 补给效果不会继续倒计时。
- 补给效果不会被清空。

当游戏回到主菜单、局外强化或结算界面时：

- 局内补给效果会清空。
- 掉落补给会清空。
- 临时装置会清空。

## 与宝箱系统的关系

局内补给不等于宝箱奖励。

这版补给系统只处理战斗中的临时道具，不处理宝箱材料。

宝箱仍应遵守：

- 宝箱不掉血包、磁铁、攻速药剂、攻击药剂、暴击药剂、冰霜炸弹、炮台、诱饵等临时补给。
- 宝箱掉材料、特殊物品、蓝图、残页、职业解锁物、区域收复物。

## 后续扩展建议

下一批可以接入：

- 净化符：清除负面状态。
- 佣兵号角：临时召唤友军。
- 亡骨号角：临时召唤骷髅。
- 丰收印记：短时间提高经验获取。
- 猎魂灯：短时间提高魂晶或材料碎片出现概率。

后续如果补给数量继续增加，建议把 `RunSupplyRuntime` 的效果逻辑拆成独立 handler，避免一个文件越来越大。
