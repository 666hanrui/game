# 局内补给系统实现说明

本文件说明当前局内补给系统的实现方式。

## 当前已接入道具

当前已实装 8 个局内补给：

- 磁铁
- 护盾
- 急速药剂
- 攻击药剂
- 血包
- 回春露
- 暴击药剂
- 冰霜炸弹

这些补给来自 `src/data/runItems.ts` 的局内补给定义。

## 实现文件

### src/systems/RunSupplyRuntime.ts

局内补给运行时系统。

职责：

- 随时间在玩家附近刷新补给。
- 击杀敌人时低概率刷新补给。
- Boss 击杀后额外刷新补给。
- 渲染补给掉落物。
- 处理玩家拾取补给。
- 管理补给持续时间。
- 渲染补给状态条。
- 渲染磁铁、护盾、回春、暴击等光环。
- 处理即时补给效果，例如血包和冰霜炸弹。

### src/main.ts

在游戏主循环外层接入 `RunSupplyRuntime`。

调用顺序：

```text
runSupply.beforeGameUpdate()
game.update(dt)
runSupply.afterGameUpdate(dt)
game.render()
runSupply.render(ctx)
```

这种方式的好处是：

- 不直接大改 Game.ts。
- 不直接大改 GameWithSound.ts。
- 不破坏现有战斗逻辑。
- 后续如果补给系统有问题，容易单独回滚。

## 道具效果

### 磁铁

持续 8 秒。

效果：大范围吸取已有经验晶体和普通掉落物。

表现：玩家周围出现蓝色吸附环。

### 护盾

拾取后获得 1 层护盾，最多 3 层。

效果：抵挡一次受到的伤害。

表现：玩家周围出现浅蓝护盾环，抵挡时出现“护盾抵挡”文字和粒子。

说明：当前护盾通过外层记录 `game.update` 前后的生命变化来抵挡伤害，因此不需要改 CombatSystem。

### 急速药剂

持续 6 秒。

效果：临时降低攻击间隔，提高攻速。

表现：右侧补给状态条显示倒计时。

### 攻击药剂

持续 6 秒。

效果：临时提高玩家伤害。

表现：右侧补给状态条显示倒计时。

### 血包

即时生效。

效果：立即恢复一段生命值。

定位：基础续航补给，玩家低血量时刷新权重会更高。

### 回春露

持续 7 秒。

效果：持续恢复生命。

表现：玩家周围出现绿色回春光环，右侧状态条显示倒计时。

### 暴击药剂

持续 7 秒。

效果：临时提高暴击率和暴击伤害。

表现：玩家周围出现金色暴击弧线，右侧状态条显示倒计时。

### 冰霜炸弹

即时生效。

效果：拾取后立即大幅减速附近敌人。

表现：玩家附近爆出冰霜粒子，并显示影响敌人数量。

定位：被围杀时的解围道具。

## 掉落和刷新逻辑

当前补给来源：

- 随时间在玩家附近刷新。
- 击杀敌人时低概率刷新。
- Boss 击杀后额外刷新护盾和药剂。
- 玩家低血量时，血包、回春露、护盾的刷新权重会提高。

## 暂停和升级阶段处理

当游戏进入升级、选择体系、选择武器、暂停阶段时：

- 补给效果不会继续倒计时。
- 补给效果不会被清空。

当游戏回到主菜单、局外强化或结算界面时：

- 局内补给效果会清空。
- 掉落补给会清空。

## 与宝箱系统的关系

局内补给不等于宝箱奖励。

这版补给系统只处理战斗中的临时道具，不处理宝箱材料。

宝箱仍应遵守：

- 宝箱不掉血包、磁铁、攻速药剂、攻击药剂、暴击药剂、冰霜炸弹等临时补给。
- 宝箱掉材料、特殊物品、蓝图、残页、职业解锁物、区域收复物。

## 后续扩展建议

下一批可以接入：

- 净化符：清除负面状态。
- 地裂符石：击退并破甲。
- 雷击符石：链式雷击清杂兵。
- 诱饵人偶：吸引敌人。
- 临时炮台：机械组预埋。
- 佣兵号角：临时召唤友军。
- 亡骨号角：临时召唤骷髅。

后续如果补给数量继续增加，建议把 `RunSupplyRuntime` 的效果逻辑拆成独立 handler，避免一个文件越来越大。
