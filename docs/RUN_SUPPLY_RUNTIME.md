# 局内补给系统第一版实现说明

本文件说明当前局内补给系统的第一版实现方式。

## 当前已接入道具

第一版只实装 4 个基础补给：

- 磁铁
- 护盾
- 急速药剂
- 攻击药剂

这些补给来自 `src/data/runItems.ts` 的局内补给定义。

## 实现文件

### src/systems/RunSupplyRuntime.ts

局内补给运行时系统。

职责：

- 随时间在玩家附近刷新补给。
- 击杀敌人时低概率刷新补给。
- Boss 击杀后额外刷新补给。
- 渲染补给掉落物。
- 处理玩家拾取补给。
- 管理补给持续时间。
- 渲染补给状态条。
- 渲染磁铁光环和护盾光环。

### src/main.ts

在游戏主循环外层接入 `RunSupplyRuntime`。

调用顺序：

```text
runSupply.beforeGameUpdate()
game.update(dt)
runSupply.afterGameUpdate(dt)
game.render()
runSupply.render(ctx)
```

这种方式的好处是：

- 不直接大改 Game.ts。
- 不直接大改 GameWithSound.ts。
- 不破坏现有战斗逻辑。
- 后续如果补给系统有问题，容易单独回滚。

## 道具效果

### 磁铁

持续 8 秒。

效果：大范围吸取已有经验晶体和普通掉落物。

表现：玩家周围出现蓝色吸附环。

### 护盾

拾取后获得 1 层护盾，最多 3 层。

效果：抵挡一次受到的伤害。

表现：玩家周围出现浅蓝护盾环，抵挡时出现“护盾抵挡”文字和粒子。

说明：当前护盾通过外层记录 `game.update` 前后的生命变化来抵挡伤害，因此不需要改 CombatSystem。

### 急速药剂

持续 6 秒。

效果：临时降低攻击间隔，提高攻速。

表现：右侧补给状态条显示倒计时。

### 攻击药剂

持续 6 秒。

效果：临时提高玩家伤害。

表现：右侧补给状态条显示倒计时。

## 暂停和升级阶段处理

当游戏进入升级、选择体系、选择武器、暂停阶段时：

- 补给效果不会继续倒计时。
- 补给效果不会被清空。

当游戏回到主菜单、局外强化或结算界面时：

- 局内补给效果会清空。
- 掉落补给会清空。

## 与宝箱系统的关系

局内补给不等于宝箱奖励。

这版补给系统只处理战斗中的临时道具，不处理宝箱材料。

宝箱仍应遵守：

- 宝箱不掉血包、磁铁、攻速药剂、攻击药剂等临时补给。
- 宝箱掉材料、特殊物品、蓝图、残页、职业解锁物、区域收复物。

## 后续扩展建议

下一批可以接入：

- 血包：局内临时回血补给。
- 回春露：持续回血。
- 暴击药剂：短时间提高暴击率和暴击伤害。
- 净化符：清除负面状态。
- 冰霜炸弹：范围减速。
- 地裂符石：击退并破甲。
- 诱饵人偶：吸引敌人。
- 临时炮台：机械组预埋。

后续如果补给数量增加，建议把 `RunSupplyRuntime` 的效果逻辑拆成独立 handler，避免一个文件越来越大。
