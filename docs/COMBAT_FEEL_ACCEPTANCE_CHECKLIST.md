# 战斗爽感与构筑系统统一验收清单

本清单用于统一验收 `COMBAT_FEEL_REWORK_PLAN.md` 的 1-7 阶段改造。

> 云端静态开发完成后，需要用户本地运行和手感测试。不要只看编译通过，重点看战斗观感、升级反馈和构筑成型感。

## 一、先跑命令

```bash
git pull
npm run check:combat-feel
npm run check:b-line
npm run typecheck
npm run build
npm run dev
```

如果前四个命令有报错，先不要测手感，直接把报错发回继续修。

## 二、基础攻击形态测试

目标：确认武器不再全都像圆形小球。

### 弓箭

测试点：

```text
1. 选择弓箭。
2. 普通攻击应是箭矢感，而不是圆球。
3. 箭矢应按飞行方向旋转。
4. 箭矢命中敌人时至少有基础命中反馈。
```

通过标准：

```text
玩家一眼能看出这是箭，不是魔法球或普通圆弹。
```

### 魔法武器

测试点：

```text
1. 选择魔杖。
2. 选择法杖。
3. 选择法球。
4. 三者 projectile 视觉应该有差异。
```

通过标准：

```text
魔杖更小更快，法杖更重，法球更偏能量感。
```

### 科技武器

测试点：

```text
1. 选择能量核心。
2. 选择无人机核心。
3. 弹体应该有科技脉冲或能量弹感觉。
```

通过标准：

```text
科技攻击不应该像普通法球，也不应该像弓箭。
```

### 近战武器

测试点：

```text
1. 选择长枪。
2. 初始攻击必须是近战刺击，不应直接发远程枪芒。
3. 选择狼牙棒。
4. 初始攻击必须是近战重击，不应直接发远程地裂。
5. 选择飞刃。
6. 飞刃可以是短距离回旋 / 投刃感，但不应该像普通圆球。
```

通过标准：

```text
长枪和狼牙棒前期有明确近战感；拿到对应升级后才出现远程派生。
```

## 三、升级卡阶段成长测试

目标：确认重复拿同一张卡不再只是小数值。

测试方法：

```text
1. 进入升级界面。
2. 看卡牌描述是否出现 当前 / 下一级 / 阶段含义。
3. 重复拿同一张卡，例如多重箭、追踪箭、枪芒、地裂、震荡波。
4. 第二次、第三次出现时，描述应能看出强化阶段变化。
```

重点卡：

```text
多重箭
追踪箭
爆裂箭
冰霜箭
枪芒
穿刺
地裂
震荡波
飞刃回旋
无人机协同
能量折射
```

通过标准：

```text
卡牌说明能让玩家明白：这次拿不是重复废卡，而是在进入下一阶段。
```

## 四、派生攻击测试

目标：确认升级卡真正改变攻击形态。

### 枪芒

测试点：

```text
1. 选择长枪。
2. 没有枪芒升级前，只能近战刺击。
3. 拿到枪芒后，刺击后应额外释放 spear_beam。
4. 多拿几次枪芒后，枪芒数量或表现应更强。
```

通过标准：

```text
长枪是先近战，再通过升级卡获得远程能力。
```

### 地裂

测试点：

```text
1. 选择狼牙棒。
2. 没有地裂前，只能近战重击。
3. 拿到地裂后，重击后应向前释放 shockwave。
4. 地裂不能被旧逻辑错误转成普通 hammer 弹体。
```

通过标准：

```text
地裂看起来像地裂波，不像小球，不像普通锤子弹。
```

### 震荡波

测试点：

```text
1. 狼牙棒拿到震荡波。
2. 重击时应出现环形冲击。
3. 清小怪时应该能感觉到范围压制。
```

通过标准：

```text
震荡波有明显范围感，而不是单体小弹。
```

## 五、天赋元素倾向测试

目标：确认元素不是升级卡，而是天赋底色。

测试前准备：

```text
进入天赋界面，装备对应元素倾向天赋。
```

需要测试的天赋：

```text
雷纹共鸣：lightning
炎脉涌动：fire
霜息凝痕：ice
毒蚀蔓延：poison
奥术星环：arcane
机巧制导：tech
```

通过标准：

```text
元素倾向应来自天赋，不应在局内升级卡池里额外出现雷卡、火卡、冰卡、毒卡。
```

## 六、隐藏联动测试

目标：确认升级卡 + 天赋会自动触发，不新增联动卡。

### 雷羽分裂

条件：

```text
装备雷纹共鸣
局内拿多重箭
```

测试点：

```text
箭矢命中后应出现电弧 / 跳电反馈。
升级面板应提示已激活或选择后联动。
```

### 制导箭

条件：

```text
装备机巧制导
局内拿追踪箭
```

测试点：

```text
追踪箭锁定范围和转向应更明显。
```

### 炎爆箭

条件：

```text
装备炎脉涌动
局内拿爆裂箭
```

测试点：

```text
命中后火焰爆裂反馈更明显。
```

### 雷贯枪芒

条件：

```text
装备雷纹共鸣
选择长枪
局内拿枪芒
```

测试点：

```text
枪芒命中后应出现电弧或跳电反馈。
```

### 寒星枪芒

条件：

```text
装备霜息凝痕
选择长枪
局内拿枪芒
```

测试点：

```text
枪芒命中后应有冰蓝反馈和减速感。
```

### 熔岩地裂

条件：

```text
装备炎脉涌动
选择狼牙棒
局内拿地裂
```

测试点：

```text
地裂命中后应有火焰 / 熔岩感反馈。
```

### 冰裂大地

条件：

```text
装备霜息凝痕
选择狼牙棒
局内拿地裂
```

测试点：

```text
地裂命中后应有冰霜反馈和减速感。
```

### 毒刃回旋

条件：

```text
装备毒蚀蔓延
选择飞刃
局内拿飞刃回旋
```

测试点：

```text
飞刃命中或回旋路径应有绿色毒蚀反馈。
```

### 奥术无人机

条件：

```text
装备奥术星环
选择无人机核心或拿无人机协同
```

测试点：

```text
无人机弹体应更偏奥术紫蓝，不只是科技蓝。
```

## 七、流派成型测试

目标：确认玩家能感受到初成 / 成型 / 大成。

测试点：

```text
1. 选择某个武器路线。
2. 连续拿该路线升级卡。
3. 左侧属性面板应显示流派名。
4. 左侧属性面板应显示阶段点数。
5. 左侧属性面板应显示隐藏联动数量。
6. 角色周围 BuildEffectOverlay 应逐步增强。
```

阶段标准：

```text
2 点左右：初成
4 点左右：成型
7 点左右：大成
```

推荐测试路线：

```text
弓箭：多重箭、追踪箭、爆裂箭、冰霜箭
长枪：枪芒、穿刺、暴击类
狼牙棒：地裂、震荡波、破甲类
飞刃：飞刃回旋、暴击类、毒蚀联动
魔法：魔杖 / 法杖 / 法球相关升级
科技：无人机、能量折射、科技弹体相关升级
```

通过标准：

```text
玩家不需要看代码，也能从左侧面板和角色周围特效看出：这个体系开始成型了。
```

## 八、杂交大成测试

目标：确认路线大成 + 隐藏联动后，会出现特殊流派名。

需要重点测：

```text
雷羽箭阵：弓箭路线成型或大成 + 雷羽分裂
熔岩地裂：钝器路线成型或大成 + 熔岩地裂联动
雷贯枪阵：长枪路线成型或大成 + 雷贯枪芒联动
毒刃风暴：飞刃路线成型或大成 + 毒刃回旋联动
奥术无人机群：科技路线成型或大成 + 奥术无人机联动
霜痕箭雨：弓箭路线成型或大成 + 冰霜箭联动
杂交大成：多个隐藏联动同时激活
```

观察点：

```text
1. 左侧属性面板流派名是否变成杂交流派名。
2. 角色周围是否出现额外彩色光环。
3. BuildEffectOverlay 是否比普通成型更明显。
4. 是否出现联动数量提示。
```

通过标准：

```text
看到名字和光效时，玩家能明显感觉“这个构筑毕业了”。
```

## 九、命中反馈测试

目标：确认打击感比以前强。

测试点：

```text
1. 普通箭矢命中：应有基础火花 / 受击反馈。
2. 魔法命中：应有紫蓝粒子或法术闪光。
3. 科技命中：应有电子 / 脉冲感。
4. 枪芒命中：应有直线贯穿感。
5. 地裂命中：应有范围震荡感。
6. 暴击命中：伤害数字和粒子应更醒目。
7. 隐藏联动命中：应出现对应元素反馈。
```

通过标准：

```text
不同攻击命中敌人时，玩家能通过视觉判断攻击类型和强度。
```

## 十、回归测试

目标：确认新系统没有把旧流程改坏。

必须测：

```text
1. 营地仍能进入远征。
2. 种族选择仍正常。
3. 体系选择仍正常。
4. 武器选择仍正常。
5. 升级界面仍能选择卡牌。
6. 暂停 / 继续 / 结算不报错。
7. Boss 波次仍正常生成。
8. 敌人击杀仍给经验。
9. 死亡结算仍保存局外资源。
```

特别注意：

```text
枪芒、地裂、震荡波这些派生攻击不能绕过 Game.ts 的击杀 / 经验结算。
```

## 十一、主观手感判断

最后用玩家视角判断：

```text
1. 现在不同武器是否一眼能分辨？
2. 近战武器前期是否真的像近战？
3. 拿到升级卡后，攻击方式是否有肉眼变化？
4. 重复拿同一张升级卡是否有期待感？
5. 天赋 + 升级卡的联动是否自然出现？
6. 左侧面板是否能看懂当前流派进度？
7. 中后期是否能明显感觉火力变爽？
8. 是否有文字太挤、特效太乱、看不清敌人的问题？
```

如果其中第 8 条出现明显问题，下一轮优先做 UI 和特效降噪。

## 十二、建议反馈格式

测试后建议按这个格式反馈：

```text
1. 命令结果：check / typecheck / build 是否通过。
2. 基础攻击：哪些武器看起来不像预期。
3. 升级面板：是否文字太挤，哪些卡描述看不懂。
4. 派生攻击：枪芒 / 地裂 / 震荡波是否正常。
5. 隐藏联动：哪些联动没触发或看不出来。
6. 流派成型：左侧面板和角色光效是否明显。
7. 杂交大成：是否出现对应名字和光环。
8. 战斗爽感：最不爽的一点是什么。
9. 截图或录屏：最好附 2-3 张关键截图。
```
